# 회고

---

처음에는 FastAPI로 어떤 구조를 잡아야 할지 몰라서, 파이썬 리스트에 데이터를 저장하는 방식으로 먼저 구현했습니다. 하지만 하다보니 데이터베이스 적용, REST API 구성, LLM 연동이 필요했기 때문에 기존 방향과는 아예 맞지 않다고 생각했습니다.

그래서 기존 파일을 그대로 사용하지 않고, week2 폴더에서 다시 구조를 잡아 구현했습니다. 게시글과 댓글 데이터를 MySQL에 저장하도록 변경했고, `GET`, `POST`, `PATCH`, `DELETE` 메서드를 사용해 게시글과 댓글 API를 구성했습니다.

추가로 Ollama 기반 LLM 모델을 연결하여 게시글 요약과 댓글 요약 기능도 구현했습니다. 이번 과제를 통해 단순 리스트 기반 저장 방식과 DB 기반 저장 방식의 차이를 알게 되었고, FastAPI 프로젝트를 Router, Controller, Model로 나누어 구성하는 방법을 연습할 수 있었습니다.
