# 회고

이번 위클리 챌린지는 교재를 참고하여 최대한 직접 코드를 작성한 뒤, AI에게 코드 리뷰를 요청하며 오류를 수정하는 방식으로 진행했습니다.

먼저 데이터 전처리 과정에서 데이터프레임 형태의 데이터를 정규화하는 방법을 검색하여 적용하였고, 또한 원-핫 인코딩을 하지 않아 데이터의 값 형태가 맞지 않는 오류가 발생해서 이를 해결하기 위해 원-핫 인코딩을 추가했습니다.

위클리 챌린지 3번 Perceptron 구현에서는 교재를 따라 작성하는 과정에서 NumPy 기반 코드에 PyTorch Tensor를 사용했는데

```python
inputs = x_train_tensor
outputs = y_train_tensor
```

Tensor는 PyTorch에서 사용하는 객체이기 때문에 NumPy 연산과 함께 사용할 경우 오류가 발생할 수 있다는 점을 알게 되었습니다. 반면 sklearn 모델은 NumPy 배열이나 pandas DataFrame을 입력으로 받을 수 있기 때문에 Tensor 변환이 필수는 아니며, Tensor 변환은 주로 PyTorch 모델을 사용할 때 필요하다는 점을 학습하였습니다.

또한 비선형 데이터셋을 만드는 방법을 AI에게 질문하여 확인했고, MLP 모델을 구현할 때는 입력 데이터를 Tensor로 변환해야 하는지, DataLoader가 필요한지도 함께 질문하였고, PyTorch 모델 학습에서는 Tensor 변환과 DataLoader 사용이 필요하다는 것을 이해한 뒤 코드에 반영했습니다.

이번 과제를 통해 단순히 코드를 따라 작성하는 것뿐만 아니라, 사용하는 라이브러리에 따라 데이터 형식이 달라진다는 점을 배웠습니다. 특히 NumPy, pandas, sklearn, PyTorch가 각각 어떤 데이터 형식을 주로 사용하는지 구분하는 것이 중요하다는 것을 느꼈습니다.
